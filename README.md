# Regime Dashboard

A local dashboard for monitoring a 3-tier macro regime strategy across **GLD**, **QQQ**, and **USO**. It scores the current regime, suggests rebalances, logs decisions to CSV, and supports **paper trading** with a Robinhood-style portfolio value chart.

No brokerage credentials are stored in this repo. Market data comes from public Yahoo Finance endpoints. Scheduled jobs run locally via macOS `launchd`.

## Strategy overview

| Tier | Regime | QQQ | USO | GLD |
|------|--------|-----|-----|-----|
| 1 | Risk-on / growth | 60% | 25% | 15% |
| 2 | Balanced / neutral | 40% | 30% | 30% |
| 3 | Risk-off / stagflation | 20% | 20% | 60% |

Rebalance suggestions cap position changes at roughly **5–10% per session**. Tier 3 favors capital preservation.

## Dashboard features

- Portfolio value, cash, and allocation vs regime target
- Drift warnings when any position exceeds a 5% band
- Market snapshot (QQQ, USO, GLD, VIX, WTI, gold, BTC)
- Regime rationale and strategy log
- **Paper mode:** mock portfolio, auto-fills on open/close jobs, P&L tracking
- **Portfolio chart:** Robinhood-style SVG line with hover crosshair, time ranges (1D / 1W / 1M / ALL), and live snapshots

## Paper trading

Paper mode is enabled in `data/paper_config.json` (gitignored). On each open/close job the engine can execute small mock trades toward the active tier target.

```bash
# Initialize / reset paper account ($100k cash, clears history)
npm run paper:init

# Full reset for a fresh start date (backs up prior logs)
npm run paper:reset
```

Paper state files live under `data/` and are **not committed**:

- `paper_config.json` — settings
- `paper_portfolio.json` — holdings and cash
- `paper_trades.csv` — trade log
- `paper_equity.csv` — open/close equity snapshots
- `paper_chart.json` — intraday chart snapshots
- `live_quotes.json` — (optional) real prices for paper mode

## Real prices from Robinhood MCP in paper mode

Paper mode normally uses public Yahoo Finance data for prices.

To use **real current prices fetched via the Robinhood MCP** (the 33 tools available in this Grok session, configured separately as noted in the original README):

1. Edit (or re-init) `data/paper_config.json` and set `"use_real_prices": true`

2. While the dashboard is running (or before), ask me in this chat:

   "sync real prices for QQQ, USO, GLD, ^VIX, CL=F, GC=F, BTC-USD using the Robinhood MCP"

   I will use the MCP tools to get live market data and either write `data/live_quotes.json` directly or POST it to the running API (`POST /api/live-prices`).

3. The server, regime jobs, portfolio calculations, rebalances, and chart will then use the MCP-sourced prices instead of Yahoo.

Example `data/live_quotes.json` (generated by Grok + MCP):

```json
{
  "QQQ": { "price": 512.34, "changePct": 0.75 },
  "USO": { "price": 72.10, "changePct": -1.2 },
  "GLD": 245.67,
  "^VIX": 18.5,
  "CL=F": 71.25,
  "GC=F": 2450.5,
  "BTC-USD": 105000
}
```

This keeps brokerage credentials and auth out of the repo while giving your paper regime strategy accurate real-market prices via the agentic Robinhood integration.

## Scheduled jobs

Jobs run **weekdays** at market open and close (Eastern Time):

| Session | Time (ET) | Purpose |
|---------|-----------|---------|
| Open | 9:30 AM | Regime score + rebalance check vs prior close |
| Close | 4:00 PM | End-of-day drift check vs same-day open |

```bash
# Install launchd agents (macOS)
npm run schedule:install

# Run manually
npm run job:open
npm run job:close
```

**XAI API key requirement (for direct LLM calls in the daily job):**

The install script reads your xAI API key from a gitignored file:

```bash
# Create the key file (one line, your actual key)
echo 'xai-yourkeyhere' > data/xai_api_key.txt
chmod 600 data/xai_api_key.txt
```

Then run `npm run schedule:install`. The script will inject the key into the LaunchAgent plists (the source templates in `scripts/launchd/` always use a placeholder and are safe to commit).

After setting up locally, re-run `npm run schedule:install` (from inside the project dir) so the LaunchAgent plists point at the correct path on this machine. Note: our local folder is named `macro` (GitHub repo name) rather than `regime-dashboard`.

Reports are written to `logs/reports/`. The strategy log is appended at `../strategy_log.csv` (parent of this repo, also gitignored).

## Security & git

**Do not commit:**

- API keys, OAuth tokens, or brokerage credentials
- Local portfolio / paper-trading state (`data/paper_*`, `data/portfolio.json`)
- Runtime logs and backups (`logs/`)
- Parent-level `strategy_log.csv`
- `data/xai_api_key.txt` (your xAI key for launchd – read by install script at install time)

`.gitignore` covers these paths. The xAI key is stored in `data/xai_api_key.txt` (gitignored) and injected into the launchd plists by `scripts/install_schedule.sh` when you run the install. The source plist templates in `scripts/launchd/` contain only a placeholder.

Robinhood MCP / Agentic account integration is configured separately in Grok — not in this repository.

## Manual holdings mode

With paper mode off, edit holdings via the dashboard UI or `POST /api/portfolio`. Positions are stored in `data/portfolio.json` (gitignored).
